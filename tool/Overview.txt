Sky: Children of the Light — Map Visualization Export Toolkit v1.0
================================================================================

【Project Overview】

This toolkit exports complete 3D data from Sky: Children of the Light maps:

· Terrain Mesh (with normals for fine bump texture)
· Scene Models (rocks, buildings, butterflies, NPCs, etc.)
· Interaction Markers (portals, meditation areas, NPC positions, etc.)

Supports all .meshes file versions (v57+ and legacy).

【File Structure】

Required Files:
├── 启动.py                  # Single map export (recommended)
├── 批量地图转换.py           # Batch export tool
├── Sky_Bstbake.py           # Core terrain parsing engine
├── meshtoobj.py             # .mesh model parser
├── bintojson.py             # .bin ↔ .json converter
└── zh字典.py                # Chinese translation table (optional)

Folders:
├── mesh/                    # Store .mesh model files
└── _meshopt/                # Windows-specific (meshopt2.dll)

【Installation】

Termux (Android):

---

pkg update && pkg upgrade
pkg install python clang cmake make binutils git
pip install lz4 meshoptimizer

Windows / Linux / Mac:

---

pip install lz4 meshoptimizer

【Usage】

1. Single Map Export (Recommended)

---

python 启动.py

Follow prompts:

1. Map folder path (contains Objects.level.bin)
2. Mesh folder path (contains .mesh files)
3. Export marker spheres? (y/n)

Output: [map_folder]/[map_name]_export/[map_name].obj

2. Batch Export

---

python 批量地图转换.py

Follow prompts:

1. Level directory path (parent of all map subfolders)
2. Mesh folder path
3. Export marker spheres? (y/n)

Output: [level_parent]/输出/[map_name]/[map_name].obj
Log: [level_parent]/输出/batch_export_timestamp.txt

3. Direct Core Script Usage

---

python Sky_Bstbake.py --unpack Dawn.meshes --export-obj

【Output Description】

The exported OBJ file contains three parts:

1. Terrain
   · Base ground mesh
   · Contains vertices (v) and normals (vn)
   · Material: terrain (grey-brown)
2. Model Instances
   · Rocks, buildings, butterflies, etc.
   · Each instance named separately
   · Transform matrix applied (position/rotation/scale)
   · Material: model (light grey)
   · Z-axis flipped (Blender-compatible)
3. Marker Spheres (optional)
   · Replace NPCs, portals, meditation areas, etc.
   · Different colors per class
   · Sphere radius: 0.5m

【FAQ】

Q1: "lz4 library not found"
A1: Run pip install lz4

Q2: "meshoptimizer module not found"
A2: Run pip install meshoptimizer
On Termux, install clang/cmake first

Q3: Terrain has 0 vertices
A3: meshopt decoding failed; check meshoptimizer installation

Q4: Models missing (.mesh not found)
A4: Extract corresponding .mesh files from game resources

Q5: meshopt decoding fails on Windows
A5: Place meshopt2.dll in _meshopt/ folder

Q6: Some maps fail during batch export
A6: Check the generated batch_export_*.txt log file

【Technical Information】

Supported Versions: LVL04 - LVL0D (v57+ fully supported)
Decoding Library: meshoptimizer (official Python bindings)
Compression: LZ4 block
Format: TGCL (BSTNodes) + GEO0 (meshopt)
